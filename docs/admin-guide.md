# BROKS · Руководство администратора

## 1. Перед стартом
1. Установить переменные окружения (`server/.env` и `.env.local`):
   ```
   MONGODB_URI=…
   CLIENT_ORIGIN=http://localhost:3000
   CDN_URL=https://cdn.broks.ru   # опционально
   NEXT_PUBLIC_API_URL=http://localhost:4000
   NEXT_PUBLIC_YANDEX_MAPS_KEY=…
   ```
2. Запустить backend:
   ```bash
   cd server
   npm install
   npm run dev   # API http://localhost:4000
   ```
3. Запустить фронтенд:
   ```bash
   npm install
   npm run dev:full   # Next + сервер
   ```

## 2. Загрузка медиа
1. **Фото** – отправить `multipart/form-data` на `POST /api/uploads/photo`:
   ```bash
   curl -X POST http://localhost:4000/api/uploads/photo \
     -F "photo=@/path/to/flat.jpg"
   ```
2. Полученный `url`/`path` сохранить – он понадобится при создании объекта.
3. Валидация:
   - форматы: JPG/PNG
   - размер ≤ 3 МБ
   - каждое объявление минимум с 1 фото

## 3. Создание объекта
1. Отправить `POST /api/properties`:
   ```bash
   curl -X POST http://localhost:4000/api/properties \
     -H "Content-Type: application/json" \
     -d '{
       "title": "БЦ \"Океан\"",
       "address": "Москва, Ленинградский пр-т, 39",
       "district": "Беговой",
       "city": "Москва",
       "cityId": 1,
       "status": "available",
       "price": 120000000,
       "pricePerMeter": 400000,
       "dealType": "buy",
       "propertyType": "apartment",
       "roomsCount": 4,
       "area": 300,
       "images": ["/uploads/property-123.jpg"],
       "developer": "ПИК"
     }'
   ```
2. Проверить ответ (`201 Created`). При ошибке – изучить `message` (чаще всего отсутствует обязательное поле).
3. Для массовой загрузки можно подготовить CSV/XML → конвертировать в JSON и отправлять поочерёдно.

## 4. Проверка карточки
1. Открыть http://localhost:3000/search, убедиться:
   - карточка появилась в каталоге;
   - фильтры по типу/городу/цены/статуса находят объект;
   - детальная страница (`/property/[id]`) содержит описание, фото, статус, тип, площадь.
2. Если надо скрыть объект – обновить status (`reserved`/`sold`) через PATCH/POST (временный костыль – удалить/обновить запись в базе).

## 5. Чеклист публикации
1. **Медиа**: ≥ 4 фото, корректные подписи.
2. **Описание**: минимум 300 символов, перечислены преимущества.
3. **Поля фильтров**: тип, город, статус, цена, площадь, этаж, год – заполнены.
4. **Контакты**: указан агент/компания.
5. **Проверка в UI**: карточка доступна, карта/галерея работает, фильтры находят объявление.

## 6. Пользовательский онбординг
- **Каталог** (`/`): кнопки "Листинги"/"Карта"/"Правила" → переключают режимы.
- **Поиск** (`/search`): расширенные фильтры, карта, сохранённые поиски.
- **Деталь** (`/property/[id]`): презентация, кнопки скачать/запросить показ.
- **Админ** (`/dashboard`): доступно после авторизации (пока mock, расширение планируется).

## 7. FAQ
| Вопрос | Ответ |
| --- | --- |
| Где лежат фото? | На сервере в `/uploads`, раздаются через `/uploads/*`. Можно подключить CDN. |
| Как обновить объект? | Временно – напрямую в Mongo (напр. MongoDB Compass) или через POST с тем же ID после удаления. |
| Почему объект не виден? | Проверь `status` (должен быть `available`), фильтры в URL и наличие обязательных полей. |
| Что делать при ошибке 3МБ? | Сжать фото, сохранить в JPEG ≤ 1600px по ширине. |
